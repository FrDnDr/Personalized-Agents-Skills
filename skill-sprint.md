# Frontend UI/UX Skill Recommendations

Adding dedicated UI/UX skills would improve frontend consistency and visual quality.

## Recommended skill files

1. `frontend/design-system.md`
   - tokens (color, spacing, radius, typography)
   - component variants (button/input/card)
   - dark mode rules

2. `frontend/accessibility.md`
   - keyboard navigation
   - focus states
   - contrast thresholds
   - semantic HTML + ARIA rules

3. `frontend/interaction-patterns.md`
   - loading/empty/error states
   - form validation UX
   - optimistic updates
   - confirmation/toast patterns

4. `frontend/responsive-layout.md`
   - breakpoint strategy
   - container widths
   - mobile-first spacing/layout rules

5. `frontend/content-ux.md`
   - microcopy style (labels, errors, CTA text)
   - truncation rules
   - localization-ready text conventions

6. `frontend/visual-hierarchy.md`
   - title/subtitle/body scale
   - spacing rhythm
   - card/list detail prioritization

## Why this helps

Current frontend skills cover structure/state well. These additions would strengthen UX quality, accessibility, and design consistency across features.
